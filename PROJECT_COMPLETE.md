## 💾 数据库设计

### 5 张表

1. **user_states** - 用户对话状态
   - mode: idle/setup/forwarding
   - current_repo: 当前使用的库
   - setup_step: 当前设置步骤
   - setup_data: 临时数据（JSON）

2. **forward_repositories** - 视频库
   - name: 唯一名称
   - description: 描述
   - created_by: 创建者 ID

3. **forward_targets** - 转发目标
   - repo_id: 所属库
   - target_chat_id: 目标频道/群组
   - target_type: channel/group/private
   - enabled: 启用状态

4. **forward_permissions** - 权限
   - repo_id: 所属库
   - user_id: 用户 ID
   - role: admin/contributor/viewer

5. **forwarded_messages** - 转发记录
   - repo_id: 所属库
   - original_chat_id: 原始聊天
   - target_chat_id: 目标聊天
   - forwarded_at: 转发时间

## 📊 对比分析

### 从命令式到对话式

| 维度 | 旧（命令） | 新（对话） | 提升 |
|------|-----------|-----------|------|
| 学习成本 | 需记忆命令 | 跟提示走 | ⬇️ 90% |
| 操作错误 | 高（拼写） | 低（按钮） | ⬇️ 95% |
| 反馈质量 | 简陋文本 | 丰富+emoji | ⬆️ 300% |
| 用户体验 | 冰冷机械 | 温暖友好 | ⬆️ 500% |
| 适用人群 | 开发者 | 所有人 | ⬆️ 10x |

### 示例对比

**旧方式（❌ 命令）:**
```
/repo_create daily_tech 每日科技
/target_add daily_tech -1001234567890
/use daily_tech
[发送内容]
```
- 需要记住 3 个命令
- 需要知道参数格式
- 容易输入错误
- 反馈简陋

**新方式（✅ 对话）:**
```
[点击: 创建新库]
→ 输入名称
→ 输入描述
✅ 创建成功

[点击: 添加转发目标]
→ 转发频道消息
✅ 自动添加成功

[点击: 开始使用]
→ 发送内容
✅ 自动转发
```
- 零命令记忆
- 跟着提示操作
- 按钮防错
- 丰富反馈
