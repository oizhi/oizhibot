# 🚀 V2 版本更新说明

## ✨ 新增功能

### 1. 🔗 Bot 邀请链接（一键添加）

**问题**：之前需要手动转发消息来添加群组，操作繁琐

**解决**：点击按钮即可在 Telegram 中选择群组

**使用流程**：
```
用户点击"添加转发目标"
→ Bot 显示邀请链接按钮
→ 用户点击按钮
→ Telegram 弹出"选择群组"界面
→ 选择群组
→ Bot 自动加入并添加为转发目标 ✅
```

**技术实现**：
- `t.me/bot?startgroup=repo_123` - Bot 邀请链接
- `my_chat_member` 事件监听 Bot 加入群组
- `pending_group_bindings` 表记录待绑定信息（10分钟过期）
- 自动识别并完成绑定

---

### 2. ⚡ 群组快捷绑定

**问题**：管理多个群组时，需要反复在私聊中操作

**解决**：直接在目标群组中发送命令即可绑定

**使用流程**：
```
在目标群组中发送：/bind 视频库名称
→ Bot 自动将该群组添加为转发目标 ✅
```

**示例**：
```
在"科技讨论群"中：
/bind 每日科技

Bot 响应：
🎉 绑定成功！
📦 视频库: 每日科技
🎯 该群组已添加为转发目标
```

**权限检查**：
- 只有视频库管理员可以绑定
- 自动验证发送权限
- 支持群组和频道

---

### 3. 💾 备份频道（自动双重保存）

**问题**：内容转发后无法追溯，缺少备份

**解决**：每个视频库可设置专属备份频道，自动保存所有内容

**使用流程**：
```
点击"备份设置"
→ 转发备份频道的任意消息
→ Bot 自动检测并配置 ✅
→ 所有内容自动双重保存（目标 + 备份）
```

**备份频道优势**：
- ✅ **永久保存**：即使删除转发目标，备份依然存在
- ✅ **便于检索**：所有内容集中在一个频道
- ✅ **导出方便**：可随时导出历史数据
- ✅ **隐私保护**：可设为私有频道，只有管理员可见

**典型场景**：
- 创建私有备份频道"每日科技-备份"
- 转发内容同时保存到公开频道和私有备份
- 公开频道可以随时调整，备份频道永久保留

## 🆕 数据库变更

### 新增表：pending_group_bindings

**作用**：临时存储 Bot 邀请链接的绑定信息

**字段**：
```sql
CREATE TABLE pending_group_bindings (
  id INTEGER PRIMARY KEY,
  chat_id INTEGER NOT NULL,          -- 群组 ID
  chat_type TEXT NOT NULL,           -- 类型（group/channel）
  chat_title TEXT,                   -- 群组标题
  repo_id INTEGER,                   -- 关联的视频库 ID
  added_by INTEGER NOT NULL,         -- 操作用户
  created_at INTEGER NOT NULL,       -- 创建时间
  expires_at INTEGER NOT NULL,       -- 过期时间（10分钟）
  UNIQUE(chat_id, repo_id)
);
```

**工作流程**：
1. 用户点击 Bot 邀请链接时，创建待绑定记录
2. Bot 加入群组时（`my_chat_member` 事件）
3. 查询待绑定记录，自动完成绑定
4. 删除待绑定记录

---

## 🔧 技术细节

### Bot 邀请链接格式

```javascript
// 生成邀请链接
const inviteLink = `https://t.me/${botUsername}?startgroup=repo_${repoId}`;

// 示例
https://t.me/your_bot?startgroup=repo_123
```

**参数说明**：
- `startgroup` - Telegram 原生参数，表示邀请到群组
- `repo_123` - 自定义参数，标识视频库 ID

---

### my_chat_member 事件

**事件触发时机**：
- Bot 被添加到群组
- Bot 被设为管理员
- Bot 被移除

**事件处理**：
```javascript
if (update.my_chat_member) {
  const newStatus = update.my_chat_member.new_chat_member.status;
  
  if (newStatus === 'member' || newStatus === 'administrator') {
    // Bot 被添加
    // 1. 查询待绑定记录
    // 2. 自动绑定到视频库
    // 3. 发送成功消息
  }
  
  if (newStatus === 'left' || newStatus === 'kicked') {
    // Bot 被移除
    // 可选：自动禁用该转发目标
  }
}
```

---

### 待绑定记录过期机制

**目的**：防止数据库积累无效记录

**实现**：
```javascript
// 创建待绑定记录时设置过期时间
const expiresAt = Date.now() + 10 * 60 * 1000; // 10分钟

// 每次处理更新前清理过期记录
await db.cleanExpiredBindings();
```

**清理策略**：
- 每次 Webhook 请求前执行
- 删除所有 `expires_at <= now` 的记录
- 不影响性能（有索引）
