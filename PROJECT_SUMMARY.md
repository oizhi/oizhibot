# 🤖 Telegram 验证机器人项目总结

## 项目概述

这是一个基于 **Cloudflare Workers + D1 数据库**构建的智能 Telegram 群组验证机器人，具备以下核心能力：

✅ **多种验证方式**：数学题、按钮选择、验证码输入  
🤖 **智能机器人检测**：10+ 维度多层次检测算法  
🛡️ **自动防护**：可疑账号自动封禁  
📊 **详细日志**：完整的检测和验证记录  
⚙️ **灵活配置**：支持群组级别的自定义配置  

## 技术栈

- **运行环境**：Cloudflare Workers (Edge Computing)
- **数据库**：Cloudflare D1 (SQLite)
- **API**：Telegram Bot API
- **语言**：JavaScript (Node.js 18+)

## 核心功能

### 1. 新成员验证流程

```
新成员加入 → 机器人检测评分 → 评分 ≥70 → 直接封禁
                                ↓ 评分 <70
                          限制权限 + 发送验证
                                ↓
                        用户完成验证（3次机会）
                                ↓
                    通过 → 解除限制 | 失败 → 封禁
```

### 2. 机器人检测算法

10个检测维度，总分100分：
- 官方bot标记 (100分)
- 垃圾关键词 (40分)
- 用户名模式 (30分)
- 快速加入 (25分)
- 数字名称 (20分)
- 无效名称 (15分)
- 无用户名 (10分)
- 无头像 (10分)
- 新账号 (5分)

### 3. 验证方式

- **数学题**：随机生成加减乘法题
- **按钮**：点击指定的 emoji
- **验证码**：输入指定单词

## 数据库设计

### 核心表

1. **verifications** - 验证记录
   - 存储用户验证状态、答案、尝试次数
   
2. **blacklist** - 黑名单
   - 封禁用户列表及原因

3. **group_configs** - 群组配置
   - 验证类型、超时时间、自动封禁等设置

4. **bot_detection_log** - 检测日志
   - 详细的检测记录，用于分析和优化

## 快速开始

```bash
# 1. 克隆/进入项目
cd telegram-verification-bot

# 2. 安装依赖
npm install

# 3. 创建 Telegram Bot
# 访问 @BotFather 创建机器人获取 Token

# 4. 登录 Cloudflare
npx wrangler login

# 5. 创建 D1 数据库
npx wrangler d1 create telegram_verification
# 记下 database_id 并更新 wrangler.toml

# 6. 初始化数据库
npm run setup-db

# 7. 设置 Bot Token
npx wrangler secret put TELEGRAM_BOT_TOKEN

# 8. 部署
npm run deploy

# 9. 设置 Webhook
# 访问 https://your-worker.workers.dev/setup

# 10. 添加机器人到群组并授予管理员权限
```

## 使用示例

### 基本使用

1. 将机器人添加到群组
2. 授予管理员权限（需要踢人、删消息权限）
3. 新成员加入时自动触发验证

### 验证流程

新成员加入 → 收到验证消息 → 点击正确选项/输入答案 → 验证通过

如果3次验证失败 → 自动踢出并加入黑名单

### 高风险账号

如果检测分数 ≥70 且开启自动封禁 → 直接踢出，不发验证消息

## 限制与改进空间

### 当前限制

1. **超时踢出**：未实现精确的超时自动踢出（需 Durable Objects）
2. **并发检测**：快速连续加入检测不够精确
3. **图片验证码**：暂不支持（可接入第三方服务）

### 建议改进

1. 使用 **Durable Objects** 实现验证超时自动踢出
2. 接入 **Cloudflare Queues** 处理异步任务
3. 添加 **KV 缓存**优化性能
4. 集成 **机器学习模型**提升检测准确率
5. 支持 **多语言界面**
6. 添加 **统计面板**（可视化检测数据）
