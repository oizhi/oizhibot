## 🔄 状态机设计

### 用户状态

```javascript
userState = {
  mode: 'idle' | 'forwarding' | 'setup',
  currentRepo: 'repo_name',
  setupStep: null | 'creating_repo' | 'adding_target' | 'granting_perm',
  setupData: {}
}
```

### 状态转换

**1. Idle（空闲）**
- 显示主菜单
- 等待用户选择

**2. Setup（设置中）**
- `creating_repo` → 等待名称 → 等待描述
- `adding_target` → 等待转发消息或 ID
- `granting_perm` → 等待用户 ID → 等待选择角色

**3. Forwarding（转发中）**
- 接收任意消息 → 自动转发
- 显示转发结果
- 提供快捷操作按钮

## 🛠 技术实现

### 1. 状态存储

使用 D1 存储用户状态：

```sql
CREATE TABLE user_states (
    user_id INTEGER PRIMARY KEY,
    mode TEXT NOT NULL DEFAULT 'idle',
    current_repo TEXT,
    setup_step TEXT,
    setup_data TEXT, -- JSON
    updated_at INTEGER
);
```

### 2. 消息路由

```javascript
async function handleUserMessage(message, state) {
  const { mode, setupStep, currentRepo } = state;
  
  // 转发模式
  if (mode === 'forwarding') {
    await forwardToRepo(message, currentRepo);
    return;
  }
  
  // 设置模式
  if (mode === 'setup') {
    await handleSetupStep(message, setupStep, state);
    return;
  }
  
  // 空闲模式 - 显示主菜单或处理命令
  await showMainMenu(message);
}
```

### 3. Callback 处理

```javascript
async function handleCallback(query) {
  const [action, ...params] = query.data.split(':');
  
  switch(action) {
    case 'repo_select':
      await showRepoDetails(params[0]);
      break;
    case 'repo_use':
      await startForwarding(params[0]);
      break;
    case 'target_add':
      await enterAddTargetMode(params[0]);
      break;
    // ...
  }
}
```

## 📋 对比总结

| 维度 | 命令式（旧） | 对话式（新） |
|------|-------------|-------------|
| 学习成本 | 高（需记忆命令） | 低（跟着提示走） |
| 操作步骤 | 少但复杂 | 多但简单 |
| 用户体验 | 专业但冷淡 | 友好且温暖 |
| 错误处理 | 报错信息 | 引导式帮助 |
| 适用场景 | 开发者 | 普通用户 |

## 🎉 总结

**核心变化:**

1. **从命令 → 对话**
   - 不需要记忆语法
   - 跟着提示一步步走

2. **从文字 → 按钮**
   - 点击代替输入
   - 减少输入错误

3. **从冰冷 → 温暖**
   - 友好的文案
   - 丰富的 emoji
   - 清晰的反馈

4. **从复杂 → 简单**
   - 每次只问一个问题
   - 提供清晰的选项
   - 保持状态连续性
